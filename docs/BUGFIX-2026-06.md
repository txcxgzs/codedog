## 2. `aiReview.fallbackReview` 敏感词级别比较永远为 false

**严重性:** 🔴 严重  
**影响:** 敏感词审核的 riskLevel 永远为 `low`，高危词未触发高风险等级

**文件:** [aiReview.js#L243-L262](file:///c:/Users/Administrator/Desktop/codedog/server/services/aiReview.js#L243-L262)

**修复:** `SensitiveWord.level` 字段在模型中定义为 `INTEGER`，但代码用 `sw.level === 'high'`（字符串）比较，恒为 false。改为数字阈值（默认 1=low, 2=medium, 3=high）：

```js
const level = Number(sw.level) || 1;
if (level >= 3) riskLevel = 'high';
else if (level === 2 && riskLevel !== 'high') riskLevel = 'medium';
```

**验证:** 在 sensitive_words 表中插入一条 `level=3` 的敏感词，重新审核应返回 `riskLevel: 'high'`, `recommendation: 'delete'`。

## 回归验证建议

1. **审核流程**
   - 在 `sensitive_words` 表插入不同 level 的敏感词，验证 fallbackReview 的 riskLevel 正确
   - 用 hCaptcha/geetest 关闭 + ai_disabled 场景回归人工审核

2. **点赞/收藏计数**
   - 并发测试：50 个用户同时点赞同一作品，期望 `praise_times` 恰好 +50（不是少或多）
   - 取消点赞应正确 -1
   - 通知收件箱收到对应消息

3. **登录限流**
   - 用同一 IP 错误密码登录 11 次，预期 11 次返回 429
   - 用不同用户名 + 同一 IP 也应能继续尝试

4. **评论回复**
   - 回复 A 的评论，@A 昵称应正确显示
   - 取消回复后点赞/取消点赞计数与后端一致

5. **举报流程**
   - 提交举报后，所有 `reviewer+` 用户应收到 `type: 'report'` 通知

6. **一致性检查**
   ```bash
   cd server && npm run check:consistency
   # expected: Consistency checks passed.
   ```

7. **服务端语法**
