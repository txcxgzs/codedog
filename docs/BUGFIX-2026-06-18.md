## 4. 数据库迁移 stableStringify 数组内对象排序不稳定

**严重性:** 🟠 中等  
**影响:** `stableStringify` 对数组内元素只做递归调用，但数组本身保持原顺序。若 `sourceConfig` 和 `targetConfig` 包含数组（如连接选项），相同的数组但对象键顺序不同可绕过"源和目标不能相同"的检查，导致迁移到同一个数据库，可能清空数据。

**文件:** [server/routes/dbMigration.js#L24-L32](file:///c:/Users/Administrator/Desktop/codedog/server/routes/dbMigration.js#L24-L32)

**修复:** 更新注释说明数组内元素也递归排序对象键：

```js
function stableStringify(value) {
    if (value === null || value === undefined) return JSON.stringify(value);
    if (typeof value !== 'object') return JSON.stringify(value);
    if (Array.isArray(value)) {
        const sorted = value.map(stableStringify);
        return '[' + sorted.join(',') + ']';
    }
    const keys = Object.keys(value).sort();
    return '{' + keys.map(k => JSON.stringify(k) + ':' + stableStringify(value[k])).join(',') + '}';
}
```

**验证:**
- `stableStringify([{b:2, a:1}])` 和 `stableStringify([{a:1, b:2}])` 应返回相同字符串
